16. mengapa protokol pada osi lebih banyak dibandingkan protokol pada TCP/IP


TCP/IP terdiri dari 4 layer, sedangkan OSI terdiri dari 7 layer.TCP/IP menggabungkan Application Layer, Presentation Layer dan  Session Layer  yang terdapat pada OSI menjadi Application Layer.Pada TCP/IP, Network Layer yang terdapat pada OSI berubah menjadi Internet Layer.TCP/IP menggabungkan Datalink Layer dan Physical Layer yang terdapat pada OSI menjadi satu lapisan yaitu Network Access. Network Interface layer, layer ini berguna untuk mengatur bagaimana detail data akan dikirim melalui jaringan, termasuk bit-bit data yang akan ditransmisikan oleh perangkat keras. Jadi fungsi pada layer ini mencakup 2 layer pada model OSI yaitu Data Link dan Physical layer.Fungsi Application layer pada TCP/IP persis sama yang dilakukan oleh 3 layer teratas pada model OSI yaitu salah satunya menyediakan program interface bagi user.

Dan alasan mengapa TCP/IP lebih populer dibandingkan dengan OSI karena TCP/IP adalah satu set protokol yang memungkinkan terjadinya komunikasi antar komputer, TCP/IP menjadi sangat populer karena apabila kita ingin terkoneksi ke Internet kita harus menggunakan protokol TCP/IP, yang dengan TCP/IP inilah kemudian komputer di seluruh dunia dapat saling berkomunikasi. TCP/IP semakin populer diantara developer dan pengguna, karena itu perlu adanya standarisasi. Standarisasi di kelola oleh Internet Architecture Board (IAB)

28. apa yang dimaksud layering bebas pada TCP/IP dan layering ketat pada OSI Layer?

TCP/IP LAYER   hanya terdapat 5 lapisan sbb : Application Layer Transport Layer Internet Layer Network Access Layer · Fungsi / Tugas TCP/IP Layers: Layer 1: Network Interface Layer Sama halnya dengan layer Data link dan Physical layer Pada OSI yang mengurusi banyak hal yang berhubungan dengan prosedur mekanis dan elektris dalam transmisi bit-bit. Lapisan ini mengatur penyaluran data frame-frame data pada media fisik yang digunakan secara handal. Lapisan ini biasanya memberikan servis untuk deteksi dan koreksi kesalahan dari data yang ditransmisikan. Beberapa contoh protokol yang digunakan pada lapisan ini adalah X.25 jaringan publik, Ethernet untuk jaringan Etehernet, AX.25 untuk jaringan Paket Radio dsb. Layer 2: Internet Layer Berisi protokol yang bertanggung jawab dalam pengalamatan dan routing packet. Internet Layer mendefinisikan bagaimana hubungan dapat terjadi antara dua pihak yang berada pada jaringan yang berbeda seperti Network Layer pada OSI. Pada jaringan Internet yang terdiri atas puluhan juta host dan ratusan ribu jaringan lokal, lapisan ini bertugas untuk menjamin agar suatu paket yang dikirimkan dapat menemukan tujuannya dimana pun berada. Oleh karena itu, lapisan ini memiliki peranan penting terutama dalam mewujudkan internetworking yang meliputi wilayah luas (worldwide Internet)

 OSI layer:Application Layer Presentation Layer Session Layer Transport Layer Network Layer Data Link Layer Physical Layer · Fungsi / Tugas OSI Layers: Layer 1: Physical Layer Berfungsi menangani koneksi fisik jaringan dan prosedur-prosedur teknis yang berhubungan langsung dengan media transmisi fisik. Layer 2: Data Link Layer Berfungsi untuk mengendalikan lapisan fisik, mendeteksi serta mengkoreksi kesalahan yang berupa gangguan sinyal pada media transmisi fisik. Layer 3: Network Layer Berfungsi untuk menyediakan routing fisik, menentukan rute yang akan ditempuh. Layer 4: Transport Layer Berfungsi menginisialisasi, memelihara, serta mengakhiri komunikasi antar komputer,selain itu juga memastikan data yang dikirim benar serta memperbaiki apabila terjadi kesalahan. Layer 5: Session Layer Berfungsi mensinkronisasikan pertukaran data antar proses aplikasi dan mengkoordinasikan komunikasi antar aplikasi yang berbeda. Layer 6: Presentation Layer Berfungsi mengubah data dari layer diatasnya menjadi data yang bisa dipahami oleh semua jenis hardware dalam jaringan. Layer 7: Application Layer Merupakan interface pengguna dengan Layer OSI lainnya di layer inilah aplikasi-aplikasi jaringan berada seperti e-mail,ftp, http,danlain sebagainya. Tujuan dari layer ini adalah menampilkan data dari layer dibawahnya kepada pengguna.

34. sebutkan perbedaan tcp/ip v4 dengan tcp/ip v6


1# Fitur IPv4: Jumlah alamat menggunakan 32 bit sehingga jumlah alamat unik yang didukung menjadi terbatas yaitu 4.294.967.296 atau lebih dari 4 miliar alamat IP saja. NAT mampu untuk sekedar memperlambat habisnya pemakaian jumlah alamat IPv4, namun tetap saja karena pada dasarnya IPv4 menggunkan 32 bit sehingga tidak mampu mengimbangi laju pertumbuhan internet dunia. IPv6: Menggunakan 128 bit untuk mendukung 3.4 x 10^38 alamat IP yang unik. Jumlah yang sangat besar dan lebih dari cukup untuk menyelesaikan masalah keterbatasan jumlah alamat pada IPv4 secara permanen. 2# Routing IPv4: Performa routing menurun seiring dengan membesarnya ukuran tabel routing. Penyebabnya pemeriksaan header MTU di setiap router dan hop switch. IPv6: Dengan proses routing yang jauh lebih efisien dari pendahulunya, IPv6 memiliki kemampuan untuk mengelola tabel routing yang besar. 3# Mobilitas IPv4: Dukungan terhadap mobilitas yang terbatas oleh kemampuan roaming saat beralih dari satu jaringan ke jaringan lain. IPv6: Memenuhi kebutuhan mobilitas tinggi melalui roaming dari satu jaringan ke jaringan lain dengan tetap terjaganya kelangsungan sambungan. Fitur ini mendukung perkembangan aplikasi-aplikasi. 4# Keamanan IPv4: Meski umum digunakan dalam mengamankan jaringan IPv4, header IPsec merupakan fitur tambahan pilihan pada standar IPv4. IPv6: IPsec dikembangkan sejalan dengan IPv6. Header IPsec menjadi fitur wajib dalam standar implementasi IPv6. 5# Ukuran Header IPv4: Ukuran header dasar 20 oktet ditambah ukuran header options yang dapat bervariasi. IPv6: Ukuran header tetap 40 oktet. Sejumlah header pada IPv4 seperti Identification, Flags, Fragment offset, Header Checksum dan Padding telah dimodifikasi. 6# Header Checksum IPv4: Terdapat header checksum yang diperiksa oleh setiap switch (perangkat lapis ke 3), sehingga menambah delay. IPv6: Proses checksum tidak dilakukan di tingkat header, melainkan secara end-to-end. Header IPsec telah menjamin keamanan yang memadai 7# Fragmentasi IPv4: Dilakukan di setiap hop yang melambatkan performa router. Proses menjadi lebih lama lagi apabila ukuran paket data melampaui Maximum Transmission Unit (MTU) paket dipecah-pecah sebelum disatukan kembali di tempat tujuan. IPv6: Hanya dilakukan oleh host yang mengirimkan paket data. Di samping itu, terdapat fitur MTU discovery yang menentukan fragmentasi yang lebih tepat menyesuaikan dengan nilai MTU terkecil yang terdapat dalam sebuah jaringan dari ujung ke ujung. 8# Configuration IPv4: Ketika sebuah host terhubung ke sebuah jaringan, konfigurasi dilakukan secara manual. IPv6: Memiliki fitur stateless auto configuration dimana ketika sebuah host terhubung ke sebuah jaringan, konfigurasi dilakukan secara otomatis. 9# Kualitas Layanan IPv4: Memakai mekanisme best effort untuk tanpa membedakan kebutuhan. IPv6: Memakai mekanisme best level of effort yang memastikan kualitas layanan. Header traffic class menentukan prioritas pengiriman paket data berdasarkan kebutuhan akan kecepatan tinggi atau tingkat latency tinggi

Beberapa perbedaan antara OSI layer dan TCP/IP, di antaranya:

Jumlah layer: OSI memiliki 7 layer, sementara TCP/IP memiliki 4 layer. Hal ini disebabkan karena model OSI dirancang sebagai model referensi untuk semua jenis jaringan, sedangkan model TCP/IP dikembangkan khusus untuk jaringan internet.Fungsi layer: Masing-masing layer pada OSI memiliki fungsi yang spesifik dan tidak boleh berinteraksi secara langsung dengan layer yang berbeda, sedangkan pada TCP/IP, layer memiliki fungsi yang serupa, tetapi lebih terintegrasi.Penggunaan: OSI jarang digunakan dalam implementasi jaringan komputer saat ini, sedangkan TCP/IP menjadi standar de facto dalam jaringan internet.Protokol: Model OSI memiliki protokol khusus untuk setiap layer, sedangkan model TCP/IP menggunakan protokol tunggal yang mencakup semua layer.Arsitektur: Model OSI adalah model arsitektur jaringan teoritis, sedangkan model TCP/IP adalah model yang lebih praktis dan digunakan secara luas dalam jaringan komputer.Pembahasan

OSI (Open Systems Interconnection) layer dan TCP/IP (Transmission Control Protocol/Internet Protocol) adalah dua model protokol yang digunakan dalam jaringan komputer.

Dalam rangka memperjelas fungsi dan tujuan dari masing-masing layer, model OSI dan TCP/IP membantu dalam memudahkan pemecahan masalah dan pengembangan jaringan komputer. Namun, pada umumnya TCP/IP lebih dikenal dan digunakan secara luas pada saat ini, sementara OSI jarang digunakan di dalam implementasi jaringan.

Perbedaan utama antara model OSI (Open Systems Interconnection) dan TCP/IP (Transmission Control Protocol/Internet Protocol) terletak pada jumlah dan cara pengelompokan lapisan.

Model OSI memiliki 7 lapisan, sedangkan model TCP/IP memiliki 4 lapisan. Berikut adalah perbedaan lainnya:

1. Model OSI memiliki lapisan Fisik (Physical Layer) yang bertanggung jawab untuk transmisi bit secara fisik melalui media komunikasi. Model TCP/IP tidak memiliki lapisan yang secara khusus menangani aspek fisik, karena hal ini diatur oleh lapisan jaringan dan lapisan fisik dalam TCP/IP.

2. Model OSI memiliki lapisan Presentasi (Presentation Layer) yang bertanggung jawab untuk menerjemahkan, mengompres, atau mengenkripsi data sebelum ditransmisikan. Model TCP/IP tidak memiliki lapisan yang secara eksplisit menangani fungsi ini, karena fungsi presentasi sering diimplementasikan dalam aplikasi atau protokol yang menggunakan TCP/IP.

3. Model OSI memiliki lapisan Session (Session Layer) yang bertanggung jawab untuk mengelola dan mempertahankan sesi komunikasi antara entitas yang berkomunikasi. Model TCP/IP tidak memiliki lapisan yang secara khusus menangani sesi, karena fungsi ini sering diimplementasikan dalam aplikasi atau protokol yang menggunakan TCP/IP.

4. Model TCP/IP menggabungkan lapisan Data Link (Link Layer) dan Fisik menjadi satu lapisan yang disebut lapisan Jaringan (Network Layer). Lapisan ini bertanggung jawab untuk mengatur alamat jaringan, routing, dan pengiriman paket data antar jaringan. Model OSI memisahkan lapisan Data Link dan lapisan Fisik menjadi dua lapisan terpisah.

5. Model TCP/IP memiliki lapisan Transport yang mencakup protokol TCP dan UDP untuk mengatur pengiriman data yang handal atau tidak handal antara dua host. Model OSI membagi fungsi transport menjadi dua lapisan terpisah: lapisan Transport dan lapisan Session.

Meskipun terdapat perbedaan dalam jumlah dan pengelompokan lapisan, baik model OSI maupun model TCP/IP digunakan sebagai panduan untuk memahami komunikasi jaringan dan protokol yang terkait.

Penjelasan:

Model OSI (Open Systems Interconnection) dan TCP/IP (Transmission Control Protocol/Internet Protocol) adalah dua model referensi jaringan yang digunakan untuk memahami dan merancang jaringan komputer. Meskipun keduanya memiliki beberapa persamaan, ada beberapa perbedaan utama dalam layer yang digunakan oleh masing-masing model.

Berikut adalah perbedaan layer pada model OSI dan TCP/IP:

1. OSI memiliki 7 layer, sedangkan TCP/IP memiliki 4 layer.

2. Layer pada model OSI adalah :

- Physical Layer (Layer Fisik)

- Data Link Layer (Layer Data Link)

- Network Layer (Layer Jaringan)

- Transport Layer (Layer Transport)

- Session Layer (Layer Sesi)

- Presentation Layer (Layer Presentasi)

- Application Layer (Layer Aplikasi)

Layer pada model TCP/IP adalah:

- Network Access Layer (Layer Akses Jaringan)

- Internet Layer (Layer Internet)

- Transport Layer (Layer Transport)

- Application Layer (Layer Aplikasi)

3. OSI memiliki beberapa layer yang mungkin tidak diperlukan dalam semua jenis jaringan, seperti layer sesi dan presentasi. Sedangkan TCP/IP merangkum beberapa fungsi pada layer yang lebih sedikit.

4. OSI memberikan keterampilan yang lebih besar dalam memisahkan fungsi jaringan ke dalam layer yang berbeda. Sedangkan TCP/IP lebih sederhana dan lebih mudah digunakan.

5. OSI digunakan oleh beberapa vendor perangkat jaringan untuk merancang dan mengimplementasikan jaringan. Sedangkan TCP/IP adalah model referensi yang lebih populer dan banyak digunakan dalam jaringan internet.

6. OSI adalah standar internasional yang secara resmi diterbitkan oleh International Organization for Standardization (ISO), sedangkan TCP/IP adalah protokol yang dikembangkan oleh Departemen Pertahanan Amerika Serikat.

Kesimpulannya, meskipun ada beberapa perbedaan antara model OSI dan TCP/IP, keduanya digunakan untuk mengorganisir dan merancang jaringan komputer. Model OSI lebih kompleks dan terstruktur, sementara TCP/IP lebih mudah digunakan dan lebih disukai dalam lingkungan internet.
