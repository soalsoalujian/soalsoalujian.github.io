7. jelaskan mengenai static routing


Static routing (Routing Statis) adalah sebuah router yang memiliki tabel routing statik yang di setting secara manual oleh para administrator jaringan. Routing static pengaturan routing paling sederhana yang dapat dilakukan pada jaringan komputer.  Menggunakan routing statik murni dalam sebuah jaringan berarti mengisi setiap entri dalam  forwarding  table  di  setiap router yang berada di jaringan tersebut.Penggunaan  routing  statik  dalam  sebuah  jaringan  yang  kecil  tentu  bukanlah suatu masalah,  hanya  beberapa  entri  yang  perlu  diisikan  pada  forwarding table di setiap router. Namun Anda tentu dapat membayangkan bagaimana jika harus melengkapi forwarding table di setiap router yang  jumlahnya  tidak sedikit dalam  jaringan yang besar.

14. Sebutkan kelebihan setting ip address dengan DHCP dibanding setting ip static?


Jawaban:

Kemudahan AdministrasiPenghematan WaktuFleksibilitasPengelolaan Pengguna yang MudahMenghindari Konflik IP

Penjelasan:

Berikut adalah beberapa kelebihan setting IP address dengan DHCP (Dynamic Host Configuration Protocol) dibandingkan dengan setting IP static:

Kemudahan Administrasi: DHCP memungkinkan administrasi jaringan yang lebih mudah. Dengan DHCP, pengaturan dan distribusi IP address secara otomatis ditangani oleh server DHCP. Ini mengurangi kerumitan dalam mengelola dan mengkonfigurasi IP address secara manual pada setiap perangkat di jaringan.Penghematan Waktu: Dengan menggunakan DHCP, pengguna atau administrator tidak perlu mengonfigurasi IP address secara manual pada setiap perangkat. Pengalokasian IP address dilakukan secara otomatis oleh server DHCP, sehingga menghemat waktu yang diperlukan untuk mengatur setiap perangkat secara individual.Fleksibilitas: DHCP memungkinkan pengalokasian IP address secara dinamis. Ini berarti bahwa perangkat dapat menerima IP address yang berbeda setiap kali mereka terhubung ke jaringan. Hal ini memudahkan penggunaan perangkat di berbagai lokasi atau jaringan yang berbeda tanpa perlu mengubah pengaturan IP secara manual.Pengelolaan Pengguna yang Mudah: Dalam lingkungan jaringan yang besar, pengelolaan perangkat dan pengguna menjadi lebih mudah dengan DHCP. Administrator dapat dengan mudah mengendalikan dan memonitor IP address yang diberikan kepada setiap perangkat yang terhubung ke jaringan melalui server DHCP sentral.Menghindari Konflik IP: Dengan menggunakan DHCP, risiko terjadinya konflik IP address dapat dikurangi. Server DHCP memastikan bahwa setiap perangkat yang terhubung ke jaringan menerima IP address yang unik, sehingga menghindari konflik IP yang dapat menyebabkan gangguan pada komunikasi dan konektivitas jaringan.

Meskipun DHCP memiliki kelebihan-kelebihan ini, terdapat juga beberapa situasi di mana penggunaan IP address static (tetap) lebih disukai, seperti ketika diperlukan kontrol penuh terhadap alamat IP yang digunakan oleh perangkat, atau ketika perangkat tersebut berfungsi sebagai server yang perlu diakses secara konsisten menggunakan alamat IP yang tetap.

24. jelaskan perbedaan static routing dan dynamic routing


Perbedaan static routing dan dynamic routing adalah :

Segi Komplektisitas

Static Routing akan semakin kompleks dalam konfigurasi seiring dengan besarnya jaringan

Dynamic Routing dalam tingkat kekompleksan untuk konfigurasinya tidak bergantung pada seberapa besar jaringan yang dibuat

Segi Penyesuaian topologi

Static routing akan memerlukan campur tangan administrator jaringan saat mengubah suatu topologi sedangkan dynamic routing akan menyesuaikan secara otomatis.

Dalam hal ini berimbas pada fakta bahwa static routing lebih cocok untuk topologi yang simple dan dynamic routing lebih fleksibel untuk menyesuaikan apakah topologi yang digunakan kompleks atau tidak.

Segi Keamanan

Static routing lebih aman daripada dynamic routing

Segi Penggunaan Resources

Static routing tidak memerlukan tambahan perangkat namun, dynamic routing membutuhkan cpu, memori dan bandwidth

Segi Alur

Static Routing memilik rute yang akan selalu sama, akan tetapi dynamic routing rutenya bergantung bagaimana kondisi dan bentuk topologinya

Pembahasan

Routing adalah suatu proses menyalurkan data - data antar komputer dalam suatu jaringan maupun antar jaringan. Biasanya data yang disalurkan berbentuk paket data. Sebenarnya, ditinjau dari jenis konfigurasinya routing dibagi menjadi tiga yaitu static routingg, dynamic routing dan minimal routing. Untuk minimal routing, konfigurasi sangat sederhana dan hanya digunakan lokal. Sedangkan berdasarkan model pengiriman dibagi menjadi routing langsung dan tidak langsung.

36. Jelaskan mengenai Static routing!


Static routing (routing statis) adalah jenis routing yang dilakukan admin/pengelola jaringan untuk mengkonfigurasi informasi tentang jaringan yang dituju secara manual .
atau bisa juga sebuah router yang memiliki tabel routing statik yang di setting secara manual oleh para administrator jaringan .

37. karakteristik dari static routing


Penjelasan:

o Static Routing mempunyai Administrative Distance (AD) 1 sehingga akan lebih dipilih dari pada dynamic routing.

o Better security, static routes tidak diadvertise dalam network.

o Use less bandwidth dari pada dynamic routing protocol, karena tidak melakukan pertukaran route.

o No CPU cycles are used to calculate and communicate routes.

o The path a static route uses to send data is known.

o Konfigurasi dan maintenance yang memakan waktu

o Tidak cocok untuk network skala besar.

o Untuk jaringan kecil yang tidak akan terjadi perubahan topologi secara significant

o Routing ke/dari stub network. Stub network adalah jaringan yang diakses hanya mempunyai 1 exit path (karena hanya mempunyai satu neighbor).

o Untuk unknown network menggunakan default route

39. Perintah untuk membuat static routing adalah A. ip route destination mask nexthop B. ip host nama host ip address C. ip route source mask nexthop D. ip route mask nexthop source E. ip route destination nexthop


Perintah untuk membuat static routing adalah ip route destination mask nexthop. Ip route adalah perintah untuk membuat static routing. Destination adalah jaringan tujuan yang ingin ditambahkan ke routing table. Mask adalah subnet mask yang digunakan dalam jaringan. Nexthop adalah address yang akan menerima paket dan memforwardnya ke jaringan remote.

Penjelasan

Static routing adalah proses meneruskan paket jaringan oleh admin atau pengelola jaringan dari satu jaringan kejaringan lainnya yang lebih spesifik. Router melakukan routing berdasarkan informasi yang diterima dari tabel routing.

40. Prinsip kerja routing static


Jawaban

Prinsip kerja = Cara kerja

Cara kerja routing statis ada 3 bagian yaitu:

1) Konfigurasi router yang dilakukan oleh admin jaringan
2) Router melakukan routing berdasarkan informasi yang diterima dari tabel routing
3) Admin Jaringan menggunakan perintah ip route secara manual untuk konfigurasi router dengan routing statis dan routing statis berguna untuk melewatkan paket data yang ada pada jaringan.

Seorang administrator harus menggunakan perintah ip route secara manual untuk mengkonfigurasi router dengan routing statis.
