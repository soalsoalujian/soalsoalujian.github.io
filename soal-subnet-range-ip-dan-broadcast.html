Subnet mask /26 memberikan 6 bit host dan 2 bit subnet, sehingga memberikan 64 kombinasi host per subnet.

Harap dicatat bahwa dalam subnetting, subnet pertama (subnet 0) dan subnet terakhir (subnet 255) biasanya tidak digunakan karena digunakan sebagai subnet khusus (subnet nol dan subnet broadcast).

5. Sebuah IP 192.168.6.∅/28 carilah:A. Jumlah subnet B. Jumlah hasil persubnetC. Block subnetD. Range IP ​


A. Jumlah subnet dapat ditemukan dengan menggunakan rumus 2^n, dimana n adalah jumlah bit yang digunakan untuk subnetting. Dalam hal ini, /28 menunjukkan bahwa 28 bit digunakan untuk alamat IP dan 4 bit digunakan untuk subnetting. Jadi, jumlah subnet adalah 2^4 = 16.

B. Jumlah hasil per subnet dapat ditemukan dengan menggunakan rumus 2^(32-n) - 2, dimana n adalah jumlah bit yang digunakan untuk subnetting. Dalam hal ini, jumlah hasil per subnet adalah 2^(32-28) - 2 = 2^4 - 2 = 14.

C. Block subnet adalah jumlah alamat IP yang tersedia dalam setiap subnet. Dalam hal ini, block subnet adalah 14 alamat IP.

D. Range IP adalah jangkauan alamat IP yang tersedia dalam setiap subnet. Dalam hal ini, range IP dapat ditemukan dengan menggunakan block subnet dan alamat IP awal dari setiap subnet. Misalnya, range IP dari subnet pertama adalah 192.168.6.0 - 192.168.6.14.

A. Jumlah subnet: Terdapat 16 subnet dalam jaringan dengan prefix length /28.

B. Jumlah hasil persubnet: Terdapat 14 host yang dapat di-assign ke setiap subnet.

C. Block subnet: Block subnet untuk jaringan ini adalah 192.168.6.0/28.

D. Range IP: Rentang IP untuk setiap subnet dalam jaringan ini adalah dari 192.168.6.1 hingga 192.168.6.14.

10. Sebuah IP 192.168.80.∅/ 2+1A. Jumlah subnet B. Jumlah hasil persubnetC. Block subnetD. Range IP ​


A. Jumlah subnet:

IP 192.168.80.0/2+1 memiliki 2^(2+1) = 2^3 = 8 subnet.

B. Jumlah hasil per subnet:

Setiap subnet memiliki 2^(32-2-1) = 2^29 = 536,870,912 alamat IP.

C. Block subnet:

Block subnet dalam IP 192.168.80.0/2+1 adalah 192.168.80.0/2+1, dimana "/2+1" menunjukkan mask subnet /2+1, yang berarti 2 bit dari 32-bit alamat IP digunakan untuk subnetting.

D. Range IP:

Range IP setiap subnet akan bervariasi tergantung pada subnet mask yang digunakan. Misalnya, jika subnet mask adalah /25, maka setiap subnet akan memiliki 128 alamat IP (dari 64 hingga 191) dan range IP akan menjadi 192.168.80.0 hingga 192.168.80.127. Namun, jika subnet mask adalah /26, maka setiap subnet akan memiliki 64 alamat IP (dari 0 hingga 63 atau dari 64 hingga 127) dan range IP akan menjadi 192.168.80.0 hingga 192.168.80.63 atau 192.168.80.64 hingga 192.168.80.127.

Untuk menjawab pertanyaan Anda, diperlukan informasi tambahan tentang subnet mask yang digunakan dalam jaringan tersebut. Subnet mask diperlukan untuk mengidentifikasi subnet, alamat IP yang valid, dan alamat IP broadcast.

Misalkan kita anggap subnet mask yang digunakan adalah 255.255.255.0 (CIDR: /24), berikut adalah jawaban untuk setiap pertanyaan:

a. Jumlah Subnet:

Jumlah subnet dapat ditentukan dengan melihat jumlah bit yang digunakan untuk subnet pada subnet mask. Dalam kasus ini, subnet mask adalah 255.255.255.0, yang berarti ada 24 bit yang digunakan untuk subnet dan 8 bit yang digunakan untuk host. Dengan 24 bit subnet, jumlah subnet yang mungkin adalah 2^24 - 2 (2^24 mengurangi subnet yang terreservasi untuk alamat jaringan dan alamat broadcast). Jadi, jumlah subnet yang mungkin adalah 2^24 - 2 = 16.777.214 subnet.

b. Alamat IP Valid:

Alamat IP yang valid dalam subnet ini adalah alamat yang digunakan untuk host. Dalam kasus ini, alamat IP 192.168.10.100 adalah alamat host, jadi itu adalah alamat IP yang valid dalam subnet tersebut.

c. Alamat IP Broadcast:

Alamat IP broadcast adalah alamat yang digunakan untuk mengirim pesan ke semua host dalam subnet. Dalam kasus ini, alamat IP broadcast dapat ditentukan dengan mengatur semua bit host pada alamat IP menjadi 1. Jadi, alamat IP broadcast untuk subnet tersebut adalah 192.168.10.255.

Untuk alamat IP 10.10.10.50/27, mari kita cari informasi terkait subnetting sebagai berikut:

1. Total IP dalam subnet:

Jumlah IP yang tersedia dalam subnet adalah 2^(32-27) = 2^5 = 32 IP.

2. Subnet mask:

Subnet mask dari alamat IP dengan prefix /27 adalah 255.255.255.224. Hal ini dapat dihitung dengan mengubah 27 bit pertama dari alamat IP menjadi 1 dan sisanya menjadi 0.

3. IP Network:

IP Network adalah alamat IP pertama dalam subnet. Untuk mendapatkan IP Network, kita akan mengubah 27 bit terakhir dari alamat IP menjadi 0. Sehingga, IP Network adalah 10.10.10.32.

4. IP Broadcast:

IP Broadcast adalah alamat IP terakhir dalam subnet. Untuk mendapatkan IP Broadcast, kita akan mengubah 27 bit terakhir dari alamat IP menjadi 1. Sehingga, IP Broadcast adalah 10.10.10.63.

5. IP Host:

IP Host adalah alamat IP yang dapat digunakan oleh perangkat dalam subnet ini. Dalam kasus ini, IP Host dapat berkisar dari 10.10.10.33 hingga 10.10.10.62, dengan 10.10.10.32 digunakan sebagai alamat IP Network dan 10.10.10.63 digunakan sebagai IP Broadcast.

Jadi, dalam subnet dengan alamat IP 10.10.10.50/27, terdapat total 32 IP yang dapat digunakan, dengan subnet mask 255.255.255.224, IP Network adalah 10.10.10.32, IP Broadcast adalah 10.10.10.63, dan alamat IP host yang dapat digunakan berkisar dari 10.10.10.33 hingga 10.10.10.62.

35. 10.11.11. 60/28 Total IP 16 Subnet mask: 255.255.255.240 IP Metwork : IP broadcast: IP host​


Penjelasan:

Untuk menghitung alamat IP jaringan (IP Network), alamat IP broadcast, dan jumlah alamat IP host dalam subnet dengan prefix /28, kita perlu menggunakan aturan subnetting.

Sebagai informasi, untuk subnet dengan prefix /28, terdapat 16 alamat IP yang dapat digunakan (dengan 2 alamat IP sudah digunakan untuk alamat jaringan dan alamat broadcast). Prefix /28 mengandung 4 bit host, sehingga kita memiliki 2^4 = 16 kombinasi alamat IP host.

1. IP Network:

Untuk menghitung alamat IP jaringan, kita perlu mengonversi alamat IP yang diberikan ke dalam bentuk biner, lalu menyisipkan 4 bit host (karena prefix /28 memiliki 4 bit host) ke kanan alamat IP. Setelah itu, kita akan mengonversi kembali alamat IP ke bentuk desimal.

Alamat IP diberikan: 10.11.11.60

Konversi ke biner: 00001010.00001011.00001011.00111100

Tambahkan 4 bit host ke kanan: 00001010.00001011.00001011.0011(0000)

Konversi kembali ke desimal: 10.11.11.48

Jadi, alamat IP jaringan (IP Network) adalah 10.11.11.48.

2. IP Broadcast:

Untuk menghitung alamat IP broadcast, kita perlu mengonversi alamat IP jaringan ke dalam bentuk biner, lalu menyisipkan 4 bit host (karena prefix /28 memiliki 4 bit host) ke kanan alamat IP. Setelah itu, kita akan mengubah 4 bit host menjadi semua bit 1, lalu mengonversi kembali alamat IP ke bentuk desimal.

Alamat IP jaringan: 10.11.11.48

Konversi ke biner: 00001010.00001011.00001011.00110000

Tambahkan 4 bit host ke kanan: 00001010.00001011.00001011.0011(1111)

Konversi kembali ke desimal: 10.11.11.63

Jadi, alamat IP broadcast adalah 10.11.11.63.

3. IP Host:

Jumlah alamat IP host dalam subnet dengan prefix /28 adalah 16. Namun, 2 alamat IP sudah digunakan untuk alamat jaringan dan alamat broadcast. Jadi, jumlah alamat IP host yang dapat digunakan adalah 16 - 2 = 14.

IP host yang dapat digunakan adalah semua alamat IP antara alamat jaringan dan alamat broadcast, yaitu dari 10.11.11.49 hingga 10.11.11.62.

Jadi, dalam subnet dengan prefix /28, terdapat total 16 alamat IP, dengan subnet mask 255.255.255.240, alamat IP jaringan adalah 10.11.11.48, alamat IP broadcast adalah 10.11.11.63, dan alamat IP host yang dapat digunakan adalah dari 10.11.11.49 hingga 10.11.11.62.
